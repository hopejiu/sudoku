# 游戏集合 — 数独模块 产品规格文档

## 1. 概述与目标

- **产品名称**：游戏集合（第一版只做数独）
- **目标**：提供干净、流畅、可自定难度的数独游戏体验，支持本地历史记录与统计。
- **平台**：Godot 4.x（跨平台，移动端 / 桌面端）
- **脚本语言**：GDScript
- **数据全部本地存储**，无用户系统，无云同步。

## 2. 核心概念

### 游戏集合 (Game Collection)
一个主界面聚合多款独立游戏的 App。**各游戏模块不共享状态、不共享存档机制、不共享生命周期**。首版仅包含数独模块。

### 数独 (Sudoku)
集合中的第一款游戏，**独立运行**，拥有自己的界面、存档、计时器、历史记录和统计系统。

---

## 3. 玩家等级与经验系统 (新增)

### 玩家等级 (Player Level)
- **经验值**通过成功通关数独获得：`每次通关经验 = 难度 + 10`。
- 等级范围 **1~无限**（无上限）。
- **每次升级所需经验固定为 100**。
- 经验值仅在通关时发放，不计失败局。
- **升级通知**：升级时弹出独立的全屏特效弹窗，含视觉动画。在通关弹窗之后展示。

### 积分系统 (Coins)
- **积分**通过成功通关获得，是商店货币。
- 通关奖励：`每次通关积分 = 难度 + 10`。
- 积分仅消耗于商店购买，不清零。

### 等级特权
- **提示次数上限**取决于玩家等级：`提示上限 = 等级 / 3 + 1`（整除，每 3 级 +1，无上限）。
- 等级提升不直接影响游戏难度选择（难度 1~16 独立）。

---

## 4. 提示系统 (修改为有限次数)

### 提示次数限制
- 每局提示次数有上限，公式：`提示上限 = 等级 % 3 + 1`（最小 1，最大 3）。
- 原 §7.3「无次数限制」规则废止，替代为本规则。
- 提示按钮在次数用完后禁用。
- 提示记录（`hint_count`）保留在历史记录中。

### 提示按钮交互
- 按钮文本显示剩余次数：`"提示 (N)"` 或 `"提示"`。
- 次数耗尽时按钮置灰/禁用。

---

## 5. 商店与道具系统 (新增)

### 商店 (Shop)
- 入口：从主界面/数独菜单进入。
- 货币：**积分**（通过通关获得）。
- 商品：多种道具，以积分购买。购买后道具进入玩家背包。

### 道具 (Items)
- 道具为**消耗品**，在游戏进行中使用。
- 使用道具**不消耗提示次数**。
- 道具效果在当格/当次操作生效。

### 道具清单

| 道具 | 效果 | 价格 | 交互方式 | 特效 |
|------|------|------|----------|------|
| **放大镜** | 永久增加本局提示次数上限 +1（本局有效，不影响后续对局） | 20 积分 | 点击 → 确认弹窗 → 生效 | 闪光特效 |
| **炸弹** | 揭露指定 3×3 宫内的所有正确答案 | 100 积分 | 拖动到目标 3×3 宫（该宫标红）→ 松开释放 | 爆炸特效 |

### 使用说明
- **放大镜**：玩家选中后弹出确认框，确认后本局提示上限永久 +1，提示按钮上的剩余次数实时更新。
- **炸弹**：玩家选中后进入"瞄准模式"，在数独网格上拖动/滑动选择对应的 3×3 宫，目标宫边框高亮为红色。松开后该宫所有格子填入正确答案。有爆炸视觉特效。

### 背包 (Inventory)
- 本地存储，每种道具计数。
- 游戏中使用道具时，背包中对应数量 -1。
- 道具数量为 0 时不可使用。

## 9. 难度系统

### 难度等级 (Difficulty Level)
- 范围 **1~16**，每级相差 **2 个挖空数**。
- 公式：`挖空数 = 26 + 等级 × 2`
- **1 级**：挖空 28 个（最简单）
- **16 级**：挖空 58 个（最难）

### 难度选择交互
- **统一滑动条**：范围 1~16。
- 展示：「难度 N」（N=1..16），**不显示挖空数**。
- 确认后立即根据**回溯法**生成唯一解题目。

## 10. 游戏界面元素

### 10.1 顶部栏
当前难度、计时器（MM:SS / HH:MM:SS，精度秒）、暂停按钮、菜单按钮（包含重置/更改难度）。

### 10.2 中部：9×9 数独网格（_draw() 渲染）
- 单个 `Control` 节点，`_draw()` 中绘制所有网格线、背景高亮、数字、笔记
- `_gui_input()` 处理点击选中、方向键导航
- **固定题目数字**（初始给出）以给定数字色显示，不可编辑。
- **用户填入数字**区域以用户数字色显示。
- **高亮当前选中格子的行、列、宫**（半透明背景 `0.08` 透明度）。
- **冲突检测**：用户填入导致行/列/宫重复时，高亮冲突格子（红色边框 `#EF4444`），**但不阻止填入**。

### 10.3 底部：自定义数字键盘
- 数字键 **1~9**。
- **删除/清除键**：见 [§12.4 清除键行为](#124-清除键行为)。
- **笔记模式切换按钮**：见 [§12.1 模式切换](#121-模式切换)。
- 桌面端同时支持**物理数字键**。

### 10.4 页面交互
- 选中已有用户填入数字的格子后，直接点击键盘数字键**覆盖替换**原数字。
- 填入数字后**焦点停留在当前格**，不移到下一个。
- 桌面端支持**方向键**（↑↓←→）在格子间导航。

## 11. 核心游戏逻辑

### 11.1 计时

| 事件 | 行为 |
|------|------|
| **开始** | 新题目生成并首次展示时开始计时 |
| **暂停 — 显式** | 用户点击暂停按钮 → 计时停止，界面只读，显示「继续」按钮 |
| **暂停 — 离开界面** | 用户返回主界面、切换游戏、进入历史等 → 自动暂停，进入暂停界面 |
| **暂停 — 切到后台** | 如来电、Home 键等 → 自动暂停计时；切回前台 → 显示暂停界面，点击「继续」恢复 |
| **结束** | 所有格子填满且无冲突时，自动停止计时，进入通关界面并记录通关 |
| **重置** | 任何重置/新局/改难度操作均使计时归零 |

### 11.2 回撤 (Undo)
- 最多记录 **50 步**，仅记录玩家填入/清除、**提示填入**的操作。笔记操作不计入。
- **无重做 (Redo)**。
- 回撤**不影响**计时和提示次数。
- 回撤按钮在界面上始终可用。
- **实现**：`Array[Dictionary]`，每条 `{"r","c","ov","nv","on","nn"}`

### 11.3 提示（受§4 限制）
- 每局提示上限取决于玩家等级：`提示上限 = 等级 / 3 + 1`（整除，每 3 级 +1，无上限）。
- **提示方式**：将当前选中格填入正确数字；若无选中则**随机选一个空格**填入。
- 提示的数字可被回撤撤销。
- 提示按钮显示剩余次数，耗尽后禁用。

### 11.4 重置、重开、更改难度
| 操作 | 行为 |
|------|------|
| **重新开始 (重置当前)** | 清空所有玩家填入的数字，保留原题目固定数字。计时归零，回撤栈清空，提示次数归零。笔记清空。 |
| **新的一局** | 当前难度下重新生成一道新题目。计时归零，回撤栈清空，提示次数归零。 |
| **更改难度** | 弹出滑动条难度选择，确认后生成新题目，所有状态重置。 |
| **注意事项** | 所有操作均需**确认对话框**（提示「当前进度将丢失」）。 |

### 11.5 自动存档与「继续上局游戏」
- **自动保存当前正在进行的游戏**（每步操作后**立即保存**完整状态，无需防抖/节流。数独操作频率低，写入性能可接受）。
- **保存内容**：盘面、计时、提示次数、回撤栈、笔记、题目信息。
- **只保存一个**未完成的游戏槽位。
- 从主界面点击「继续上局游戏」加载该状态，继续对局。
- 如果已通关则「继续」按钮**置灰或隐藏**。
- **实现方式**：`FileAccess.store_var` 二进制序列化，`SaveManager` Autoload 管理。三文件分离：`settings.save` / `current.save` / `history.save`

## 12. 候补数字 / 笔记模式 (Pencil Marks)

### 12.1 模式切换
底部键盘区域增加**「笔记模式」切换按钮**。
- **正常模式**（默认）：点击数字键**直接填入**格子。
- **笔记模式**：点击数字键在格子上**切换候选数字**（开/关）。

### 12.2 多候选支持
一个格子可以标记**多个候选数字**（1~9 的子集），点击同一个数字切换开/关。候选数字以**小号字体**在格内显示。

### 12.3 与数字的互斥关系
笔记和数字**互不干涉**：
- 在带笔记的空格填入数字 → 笔记**保留**。
- 在有数字的格子也可进行笔记操作。

### 12.4 清除键行为
- **正常模式**：清除键删除用户填入的数字，**不影响笔记**；空格无操作。
- **笔记模式**：清除键**清空**当前格所有候选数字。

### 12.5 回撤
笔记操作**不**计入回撤栈。

## 13. 通关后行为

### 13.1 通关弹窗
- 通关后弹窗显示：**通关用时**、**提示使用次数**、**获得经验值**、**获得积分**。
- 弹窗**必须三选一**，不可关闭：
  1. **降低难度**（等级减 1，不低于 1，生成新题目）
  2. **维持难度**（同难度新题目）— **默认选中，高亮显示**
  3. **增加难度**（等级加 1，不超过 16，生成新题目）
- **边界处理**：
  - 在 **1 级**时「降低难度」按钮**禁用**。
  - 在 **16 级**时「增加难度」按钮**禁用**。
- 选择后自动开始新对局，计时归零。

### 13.2 连续通关鼓励
- 针对**同难度**记录通关连胜次数。
- 每连续通关 **5 次**（同难度）触发一次鼓励话术弹窗，引导尝试更高难度。
- 触发后连胜计数**归零重新计算**（可循环触发）。
- **打断条件**：中途失败、重置、更改难度、选择「降低难度」→ 计数归零。

### 13.3 通关奖励
- **经验值**：通关时发放。公式：`经验 = 难度 + 10`。
- **积分**：通关时发放。公式：`积分 = 难度 + 10`。
- 高难度通关奖励相同数值（难度越高奖励越高）。
- 升级所需经验固定为 100，不随等级增长。

## 14. 历史记录与统计信息

### 14.1 存储内容
最多保存最近 **20 局**（元数据 + 完整游戏状态，支持继续）。
每一局包含：
- 日期时间
- 难度等级
- 总用时（秒）
- 是否通关
- 提示使用次数
- 完整盘面快照（固定数字 + 用户最终填入数字 + **每个格子的候选数字/笔记**）

### 14.2 展示
- 列表按**时间倒序**，每行显示：日期、难度、用时、通关标记。
- 点击某行进入**详情页**（只读盘面，含笔记）。

### 14.3 详情页操作
| 按钮 | 行为 |
|------|------|
| **重开** | 加载该局的**原始题目**（无任何填入，计时归零，提示次数归零，笔记清空）作为新游戏，**覆盖**当前正在进行的游戏 |
| **继续**（仅未完成局） | 加载该局**完整存档**（盘面、计时、提示次数、回撤栈、笔记），**覆盖**当前游戏 |

### 14.4 统计摘要
- 总游戏次数（所有难度累计）
- 通关次数及通关率
- 各难度下的**最佳用时**（最快通关时间）
- 各难度下的**平均用时**（仅计算通关对局）

## 15. 约束与限制

- **固定题目数字**不可通过任何操作（回撤、提示、重置）修改。
- **冲突检测**仅视觉提示，**不阻止游戏胜利判定**。`conflict[9][9]` bool 数组标记冲突格，每次操作后全盘扫描更新。
- **胜利条件**：所有 81 格填满且无冲突。
- 生成的题目必须保证**唯一解**（`SudokuSolver` 回溯法验证）。

## 16. 第一版范围

### 包含
- 数独完整游戏（难度 1~16、计时、回撤、提示、笔记、自动存档）
- 2 种主题（经典、紫色浅色）
- 历史记录与统计（最近 20 局）
- 玩家等级与经验系统
- 积分系统与商店
- 道具系统（放大镜、炸弹）

### 不做
- 音效（预留开关，后续扩展）
- 其他游戏类型
- 云同步、用户系统

## 17. 术语表 (Glossary)

| 术语 | 定义 |
|------|------|
| **Godot 4.x** | 项目使用的游戏引擎 |
| **GDScript** | 项目使用的脚本语言，Godot 原生 |
| **场景 (Scene)** | Godot 的基本组织单元，对应 `.tscn` 文件 |
| **Autoload** | Godot 全局单例机制，注册的脚本/场景在项目启动时自动加载，全局可访问 |
| **存档管理器 (SaveManager)** | Autoload 单例，负责所有数据持久化读写。多文件分离：`settings.save` / `current.save` / `history.save` |
| **主题管理器 (ThemeManager)** | Autoload 单例，管理全局主题切换与配色数据。标准 UI 通过 Godot `Theme` 资源驱动，数独网格专属颜色通过 ThemeManager 字典读取 |
| **盘面数据** | 扁平数组三件套：`grid[9][9]` (int, 0=空)、`given[9][9]` (bool)、`notes[9][9]` (int 位掩码 1<<1~1<<9)，封装在 `SudokuBoard.gd` 中 |
| **谜题生成器** | 纯 GDScript 类 `SudokuGenerator`、`SudokuSolver`，静态方法，无场景/节点依赖 |
| **回撤栈 (Undo Stack)** | `Array[Dictionary]`，每条 `{"r","c","ov","nv","on","nn"}`，最多 50 条 |
| **冲突检测** | 全盘扫描，`conflict[9][9]` bool 数组，每步操作后 `update_conflicts()` |
| **_draw() 渲染** | 数独网格使用 Godot `Control._draw()` 自定义绘制，单节点无子节点，通过 `_gui_input()` 处理交互 |
| **场景切换 (Scene Switching)** | 使用 `SceneTree.change_scene_to_file()` 在不同游戏模块之间切换，Main 与子游戏独立，旧场景完全卸载 |
| **弹窗覆盖层 (Overlay Panel)** | 自定义 Control Panel，在场景中内嵌为子节点，通过 `visible` 属性控制显示/隐藏，不影响场景树结构 |
| **玩家等级 (Player Level)** | 全局等级，通过通关获得经验值提升。影响提示次数上限。 |
| **积分 (Coins)** | 商店货币，通过通关获得，用于购买道具。 |
| **商店 (Shop)** | 用积分购买道具的界面。 |
| **背包 (Inventory)** | 玩家拥有的道具列表及数量，本地持久化。 |
| **道具 (Item)** | 消耗品，在游戏中使用的辅助工具。包括橡皮擦、透视镜、放大镜、炸弹。 |
| **提示上限 (Hint Cap)** | 每局可使用的提示次数上限。公式：`等级 / 3 + 1`（整除）。 |
